## ✨ 项目简介

### 📖 这是什么？

**乡村教师智能教学助手**是一款专为乡村教师打造的 **Web 端 AI 教学辅助工具**，旨在解决教育资源不均衡问题，帮助教师在有限条件下高效备课、智能生成教学资源。

### 🎯 解决什么痛点？

| 痛点 | 我们的解决方案 |
|------|--------------|
| ❌ 备课耗时长，资源匮乏 | ✅ AI 一键生成完整课件/教案 |
| ❌ 重难点不知怎么讲 | ✅ 多维度拆解 + 可视化思维导图 |
| ❌ 出题费时间 | ✅ 自动生成练习题 + 难度分级 |
| ❌ 课堂语言组织难 | ✅ 生成完整讲解脚本（含话术） |
| ❌ 缺少可复用模板 | ✅ 内置 8 大学科资源库 |

### 👥 目标用户

- 🏫 **乡村中小学教师**：资源有限、教研支持不足的一线教育工作者
- 📚 **偏远地区教师**：缺乏专业培训机会的教师
- 🎯 **新任教师**：需要快速上手备课的年轻教师
- 💡 **所有教师**：希望提升备课效率的教育工作者

### 💡 产品价值

1. **降低备课负担**：从数小时缩短到几分钟
2. **提供可落地方法**：生成的内容直接可用或稍作调整
3. **随时响应的"搭档"**：7×24 小时在线，无需等待
4. **促进教育公平**：让优质教育资源触手可及

## 🚀 核心功能

### 1️⃣ AI 智能备课中心 📝

> **一键生成完整的课件和教案**

**功能特点：**
- 📋 **分步向导式操作**：基本信息 → 教学目标 → 参数配置（3 步完成）
- 🎨 **4 种教学风格可选**：
  - 📖 正式讲授型（严谨系统）
  - 🎨 故事化教学（生动有趣）
  - 🔬 互动探究型（启发思考）
  - ✨ 简洁明了型（直击要点）
- 📊 **智能内容生成**：
  - 完整的教学流程设计
  - 各环节时间分配建议
  - 重点难点标注
  - 板书设计和作业布置
- 📤 **多格式导出**：Word / PDF / 打印

**适用场景**：日常备课、公开课准备、教学比赛

---

### 2️⃣ 重难点拆解工具 🔍

> **深度分析知识点，让复杂变简单**

**功能特点：**
- 🧠 **三栏式布局**：
  - 左侧：输入面板（知识点 + 学科选择）
  - 中间：交互式思维导图可视化
  - 右侧：详细分析面板
- 🗺️ **4 大分析维度**：
  - 📚 核心概念（定义 + 通俗解释 + 形象类比）
  - 💡 实例演示（3 道典型例题）
  - ⚠️ 易错点提醒（常见错误 + 正确做法）
  - 🚀 进阶应用（竞赛题型、跨学科联系）
- 🎨 **动态主题色**：根据选择的学科自动切换配色方案
- 📊 **SVG 思维导图**：支持点击展开、悬停高亮

**适用场景**：备课前的知识梳理、学生答疑、复习课设计

### 3️⃣ 课堂练习工坊 ✏️

> **自动出题，秒生成练习卷**

- 📦 **批量操作**：全选、批量导出、打印

**适用场景**：课堂小测、单元测试、作业布置、考前冲刺

## 📘 使用指南

### 🎯 场景一：快速生成一份数学教案

**目标**：为初二学生准备"一元二次方程"的教案

**操作步骤**：

1. **打开网站** → 点击首页的 **"智能备课"** 卡片
2. **填写基本信息**：
   - 课题名称：`一元二次方程`
   - 选择学科：`数学`
   - 选择年级：`初二`
3. **设置教学目标**（示例）：
   ```
   理解一元二次方程的定义和一般形式
   掌握因式分解法求解一元二次方程
   能够应用韦达定理解决相关问题
   ```
4. **配置参数**：
   - 课时长度：`45分钟`
   - 教学风格：`互动探究型`
   - 学生水平：`中级`
   - ✅ 勾选"包含课堂练习"
5. **点击"开始生成"** → 等待 2-3 秒
6. **查看结果**：
   - 左侧目录树浏览结构
   - 右侧查看详细内容
   - 高亮显示重点内容
7. **导出使用**：
   - 点击 **"导出 Word"** 下载 .doc 文件
   - 或点击 **"打印"** 直接打印

**预期效果**：生成包含导入→讲授→练习→总结的完整教案，可直接用于课堂教学！

---

### 🎯 场景二：拆解一个物理难点

**目标**：帮助学生理解"牛顿第二定律"

**操作步骤**：

1. 进入 **"重难点拆解"** 页面
2. 输入知识点：`牛顿第二定律`
3. 选择学科：`物理`
4. 点击 **"开始分析"**
5. **查看四维分析**：

   **📚 核心概念 Tab**
   - 正式定义：F=ma，物体的加速度与所受合力成正比...
   - 通俗理解：就像推购物车，用力越大跑得越快...
   - 形象类比：把力想象成"推动者"，质量是"懒惰程度"...

   **💡 实例演示 Tab**
   - 例题1：基础计算（已知 F 和 m，求 a）
   - 例题2：综合应用（斜面问题）
   - 例题3：生活实例（汽车加速）

   **⚠️ 易错点提醒 Tab**
   - 错误1：混淆质量与重力（m vs mg）
   - 错误2：忽略力的矢量性
   - 错误3：单位换算错误
   - （每个错误都有正确做法和记忆口诀）

   **🚀 进阶应用 Tab**
   - 竞赛题型：连接体的处理
   - 跨学科：与生物（运动生理学）的联系
   - 拓展资源：推荐视频和书籍

6. **保存或打印**分析结果

**预期效果**：从多个角度彻底搞懂这个知识点！

### 🎯 场景三：快速生成一套英语练习题

**目标**：为高一学生准备 10 道"定语从句"练习题

**操作步骤**：

   - 学科：`英语`
   - 年级：`高一`

   - 难度分布：

     - 中等：50%（5道）
     - 困难：20%（2道）

   - ✅ 包含答案解析

   - ✅ 混合题型

6. **查看题目列表**：

   - 点击 **"显示答案"** 展开解析
   - 可以单独编辑某道题
7. **批量导出**：
   - 全选 → 导出 Word → 打印分发给学生

**预期效果**：10 分钟内完成原本需要 2 小时的出题工作！

## 🔧 如何接入真实 AI 服务

### ⚠️ 重要提示

**当前版本使用模拟数据**（无需 API Key），适合演示和体验完整功能。

如需获得真正的 AI 智能生成能力，请按照以下步骤配置：

---

### 方案 A：OpenAI GPT API（效果最佳）

#### ① 获取 API Key

1. 访问 [OpenAI Platform](https://platform.openai.com/)
2. 注册/登录账号
3. 进入 **API Keys** → **Create new secret key**
4. 复制生成的 Key（格式：`sk-xxxxx`）

#### ② 安装依赖并配置

```bash
# 安装 OpenAI SDK
npm install openai
```

创建 `.env` 文件（项目根目录）：

```env
VITE_OPENAI_API_KEY=sk-your-api-key-here
VITE_OPENAI_BASE_URL=https://api.openai.com/v1
```

#### ③ 切换到真实 AI 服务

```bash
# 备份原文件
cp src/data/services/aiService.ts src/data/services/aiService.mock.ts

# 使用真实版本（已为你准备好）
cp src/data/services/aiService.real.ts src/data/services/aiService.ts

# 重启开发服务器
npm run dev
```

#### ④ 成本估算

| 模型 | 单次请求 | 月费用（1000次）|
|------|---------|----------------|
| GPT-4o-mini | ~¥0.007 | ~¥7 |
| GPT-4o | ~¥0.07 | ~¥70 |

**推荐**：使用 `gpt-4o-mini`，性价比最高！

**优势**：国内访问快、中文能力强、有免费额度

1. 访问 [阿里云百炼平台](https://dashscope.console.aliyun.com/)
2. 登录阿里云账号

#### ② 安装依赖

```bash
npm install @dashscope/sdk
```

#### ③ 配置环境变量


### 🔄 快速切换开关（推荐实现）

为了方便在模拟模式和真实 AI 之间切换，你可以：

**修改 `src/data/services/index.ts`**（新建此文件）：

```typescript
// 配置模式：'mock' | 'openai' | 'qwen' | 'ollama'
const AI_MODE = import.meta.env.VITE_AI_MODE || 'mock';

export * from './aiService'; // 始终导出接口类型

if (AI_MODE === 'openai') {
  module.exports = require('./aiService.real');
} else if (AI_MODE === 'qwen') {
  module.exports = require('./aiService.qwen');
} else {
  // 默认使用模拟数据
  module.exports = require('./aiService.mock');
}
```

然后在 `.env` 中设置：
```env
VITE_AI_MODE=mock  # 或 openai / qwen / ollama
```
