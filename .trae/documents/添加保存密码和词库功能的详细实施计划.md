# 添加保存密码和词库功能的详细实施计划

## 一、保存密码功能

### 功能需求
- 登录页面添加"记住密码"选项
- 安全存储用户登录凭证
- 应用启动时自动登录
- 支持手动清除保存的密码

### 技术实现

#### 1. 安装依赖
```bash
npx expo install expo-secure-store
```

#### 2. 修改LoginScreen.js
- 添加"记住密码"开关组件
- 保存用户选择的登录状态
- 登录成功后保存凭证到安全存储

#### 3. 修改SplashScreen.js
- 检查安全存储中的登录凭证
- 如有有效凭证，自动登录

#### 4. 实现AuthService
- 创建`src/services/authService.js`
- 封装安全存储的操作方法

#### 5. 代码修改点

**LoginScreen.js**
- 添加`rememberPassword`状态
- 添加`Switch`组件
- 登录成功后调用`authService.saveCredentials()`

**SplashScreen.js**
- 在`checkLoginStatus`中调用`authService.getCredentials()`
- 如有凭证，自动登录

## 四、预期效果

### 保存密码功能
- 登录页面显示"记住密码"开关
- 勾选后，下次启动应用自动登录
- 密码安全存储，无法被恶意获取
- 支持在设置中清除保存的密码

### 词库功能
- 可创建多个词库，如"大学英语四级"、"商务英语"等
- 支持从CSV/JSON文件导入词库
- 可切换不同词库进行学习
- 支持词库的导出和分享
- 词库数据与现有学习系统无缝集成

## 五、技术风险评估

### 保存密码功能
- 风险：密码存储安全性
- 对策：使用`expo-secure-store`进行加密存储

### 词库功能
- 风险：大文件导入性能
- 对策：实现异步导入和进度显示
- 风险：不同格式文件的兼容性
- 对策：添加格式验证和错误处理
